① Discovery
interview-me Discovery
Extracts what the user actually wants via one-question-at-a-time interview until ~95% confidence. Closes the gap before any plan, spec, or code exists.
reads
user conversation (live)
writes
docs/discovery/interview-notes.md
② Product
prd Product
Generates PRD with executive summary, user stories, technical specs, and risk analysis. Output format adapts by target: Stakeholder / Dev+Designer / AI Agent.
reads
docs/discovery/interview-notes.md
writes
docs/prd.md
③ Strategy — parallel
runs simultaneously ↓
ux-strategist Strategy
Defines problem, maps user flows, structures IA, edge cases, heuristics. Never touches colors or copy.
reads
docs/prd.md
writes
docs/design/ux-blueprint.md
ux-writer Strategy
Writes microcopy, labels, error messages, CTAs, empty states. Clarity-first, user-tested language.
reads
docs/prd.md docs/design/ux-blueprint.md
writes
docs/design/copy.md
④ Design System (greenfield only)
ds-builder Design System if no DS exists
Creates complete design system from scratch — tokens, component shells, UI patterns, and design docs. Foundation for everything downstream.
reads
docs/prd.md brand brief (optional)
writes
design-system/design.md design-system/components.json design-system/ui.json design-system/patterns.json
design-component-builder DS
Builds detailed atom/molecule/organism specs with variants, props, states, and responsive rules. Output: JSON manifest.
reads
design-system/design.md
writes
design-system/components.json
design-ui-builder DS
Assembles page layouts, sections, and screen flows from components. Dual-mode: iframe (iteration) / inline (dev handoff).
reads
design-system/components.json design-system/design.md
writes
design-system/ui.json
⑤ UI Specification — parallel
ui-ux-pro-max Style Intel
Design intelligence layer — 50+ visual styles, 161 palettes, 57 font pairings. Provides style direction and aesthetic decisions before pixel work.
reads
docs/prd.md style context (in-conversation)
writes
design-system/design.md in-context style decisions
ui-implementation-specialist UI Spec
Maps UX Blueprint → DS components + tokens. Every component named, every token referenced. Dev-ready spec with Figma node IDs and a11y notes.
reads
docs/design/ux-blueprint.md design-system/components.json design-system/design.md Figma (MCP)
writes
docs/design/ui-spec.md
⑥ Spec
spec-driven-development Spec
Writes structured spec before any code. 4-phase gated workflow: Specify → Plan → Tasks → Implement. Each phase needs human review before advancing.
reads
docs/design/ui-spec.md docs/prd.md
writes
docs/specs/[feature].spec.md tests/ (test stubs)
⑦ Engineering
frontend-ui-engineering Engineering
Builds production-quality UI — accessible, performant, visually polished. Composable components, proper token usage, no "AI aesthetic". Colocated tests + stories.
reads
docs/design/ui-spec.md docs/specs/[feature].spec.md docs/design/copy.md design-system/components.json design-system/design.md
writes
src/components/[Name]/ src/pages/ src/styles/tokens.css
if bug found → back to frontend-ui-engineering
⑧ Testing
browser-testing-with-devtools Testing
Live browser verification via Chrome DevTools MCP. Reads DOM, console logs, network requests, performance traces. Sees exactly what the user sees — no guessing.
reads
localhost (running app) DOM · console · network · perf
writes
docs/testing/[date]-results.md
⑨ Launch
shipping-and-launch Launch
Pre-launch checklist across code quality, security, performance, accessibility, infrastructure, and docs. Feature flag strategy, staged rollout, rollback plan.
reads
docs/testing/[date]-results.md docs/prd.md docs/specs/[feature].spec.md
writes
CHANGELOG.md feature flag config CI/CD trigger
